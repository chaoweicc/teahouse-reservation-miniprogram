📝 项目简介
开发背景：解决传统线下 / 微信私信预定效率低、时段冲突、订单管理混乱等痛点，提升茶室运营效率与用户体验。
开发角色：独立负责全流程开发（需求梳理 → UI 设计 → 前端开发 → 后端搭建 → 测试交付）
开发周期：2 周
上线状态：已交付茶室运营方正式使用
✨ 核心功能
用户端
微信授权登录：一键授权微信登录，免注册快速使用，降低用户门槛
房型可视化展示：展示茶室房型信息（图片、容纳人数、价格、配套设施）
智能时段预定：支持按日期 / 时段选择，实时校验时段冲突，避免重复预定
全链路订单管理：订单提交、状态查询、取消订单，操作流程清晰
微信消息通知：预定成功 / 超时取消自动推送微信服务通知，及时触达用户
管理端
订单审核处理：查看 / 确认 / 取消用户预定订单
房型信息维护：新增 / 编辑房型信息、价格、库存
经营数据统计：查看预定数据、营收趋势，辅助运营决策
🛠️ 技术栈
前端：微信小程序原生开发（WXML/WXSS/JavaScript）、Vant Weapp 组件库、Flex 响应式布局
后端：微信云开发（云数据库 / 云函数 / 云存储），无服务器架构，零运维成本
工具：微信开发者工具、Git/GitHub、Postman（接口调试）
